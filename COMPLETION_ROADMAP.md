# EVICS Completion Roadmap

**System:** IAGT AI Viral Engine (EVICS)
**Store:** iamgenesistech.myshopify.com
**Goal:** Fully operational, revenue-generating, elite AI marketing intelligence system

---

## Current State Summary

The EVICS system is approximately 60% complete. The dashboard renders correctly, the backend has 23 working API endpoints, Supabase integration is wired, and three video generation platforms (HeyGen, Runway, Kling) are integrated at the code level. The primary gaps are: missing environment variables blocking live data, one missing utility file crashing Shopify routes, an incomplete agent engine layer, and six external API integrations at placeholder status.

---

## Phase 1 — Core Fixes & Live Data (Week 1)

**Goal:** Get the system running on live data with zero crashes.

### 1.1 Set Environment Variables in Railway

Set the following in Railway dashboard → Environment:

| Variable | Value |
|---|---|
| `SUPABASE_URL` | Your Supabase project URL |
| `SUPABASE_KEY` | Your Supabase anon or service role key |
| `REACT_APP_SUPABASE_URL` | Same Supabase project URL (for frontend) |
| `REACT_APP_SUPABASE_ANON_KEY` | Your Supabase anon key (for frontend) |
| `SHOPIFY_STORE_DOMAIN` | `iamgenesistech.myshopify.com` |
| `SHOPIFY_ADMIN_ACCESS_TOKEN` | Your Shopify Admin API token |
| `HEYGEN_API_KEY` | Your HeyGen API key |
| `RUNWAY_API_KEY` | Your Runway ML API key |
| `KLING_API_KEY` | Your Kling AI API key |
| `PORT` | `3000` |
| `NODE_ENV` | `production` |

**Deliverable:** Dashboard switches from Demo mode to live Supabase data on next deploy.

---

- `fetchShopifyProducts()` — calls `GET /admin/api/2024-01/products.json` and returns a normalized product array

**Deliverable:** `/api/shopify/products` and `/api/shopify/collections` routes work without crashing.

3. Return `{ success: true, products: [...] }`.

**Deliverable:** Product Matching panel loads live Shopify products.

## Phase 2 — Agent Intelligence (Weeks 2–3)

**Goal:** Build the missing engine layer so the system makes intelligent, profit-driven decisions automatically.

### 2.1 Wire OpenAI GPT-4o into Copilot

Add `OPENAI_API_KEY` to Railway. Update `/api/agent/copilot` in `server.js` to call `openai.chat.completions.create()` with a system prompt encoding EVICS rules. Keep the existing rule-based fallback for when the key is absent.

Set `window.IAGT_FEATURES.externalApis.openai = true` in `config.js`.

**Deliverable:** Copilot answers are generated by GPT-4o with full workspace context.

---

### 2.2 Build Profit Auditor Engine

Create `engines/profitScoreEngine.js`. Connect to Shopify Orders API to pull revenue and refund data. Calculate net profit per SKU. Write daily profit logs to Supabase. Add a backend route `POST /api/agent/profit-audit` to trigger on demand.

**Deliverable:** Daily profit scores available in Supabase. Capital allocator has data to work with.

---

### 2.3 Build Product Tier Manager

Create `engines/productTierEngine.js`. Implement the 4-tier ranking system using profit scores from the Profit Auditor. Add weekly re-ranking logic and Tier 4 recovery timer (2-month cutoff). Add route `GET /api/agent/product-tiers`.

**Deliverable:** All products have tier assignments. Dashboard can show tier badges on product cards.

---

### 2.4 Build Capital Allocator

Create `engines/capitalAllocatorEngine.js`. Implement the 80/20 budget rule: 80% to Top 30 ads, 20% to Promotion Pool. Add route `POST /api/agent/allocate-budget` that returns daily allocation recommendations.

**Deliverable:** Daily budget allocation report available. Operator can follow recommendations or automate.

---

### 2.5 Wire Sidebar Navigation

Update `app.js` to add `state.activeSection` and conditional section rendering. Wire all 6 sidebar buttons. Build out the AI Reconstruction, Distribution, and Analytics sections with real data.

**Deliverable:** Full dashboard navigation works. All 6 sections are accessible.

---

### 2.6 Add Rate Limiting

Install `express-rate-limit`. Apply limits to all `/api/agent/*` and `/api/video/generate` routes. Add appropriate error responses (HTTP 429 with retry-after header).

**Deliverable:** System is protected against runaway loops and quota exhaustion.

## Phase 3 — Elite Features (Weeks 4–5)

**Goal:** Add the features that separate EVICS from a basic dashboard and make it a genuine competitive advantage.

### 3.1 Real-Time Render Status Polling

Add a polling loop in `app.js` that checks `evics_renders` every 10 seconds for pending jobs. For each pending job, call the platform status endpoint and update the render card when complete.

**Deliverable:** Video renders update from "Pending" to "Complete" automatically without page refresh.

---

### 3.2 ROAS Tracking

Connect Shopify order data to creative IDs via UTM parameters. Calculate ROAS per creative. Display on creative cards. Feed into Weighted Profit Score.

**Deliverable:** Every creative has a measurable ROAS. The system knows which ads are actually making money.

---

### 3.3 Build Render Grading Engine (HAVE System)

Create `engines/renderGradingEngine.js`. Implement automated quality scoring across 6 dimensions (facial realism, hand realism, motion, lighting, behavior, AI artifacts). Enforce minimum deployment score of 92. Flag renders below threshold for human review.

**Deliverable:** No low-quality renders reach the publishing queue automatically.

---

### 3.4 Build Experiment Governor

Create `engines/experimentGovernorEngine.js`. Implement A/B test management with statistical confidence scoring. Add route `GET /api/agent/experiments` to view active tests and results.

**Deliverable:** Every creative format is tested against a baseline. Winners are promoted automatically.

---

### 3.5 Build Library Steward

Create `engines/libraryStewardEngine.js`. Implement nightly cleanup: archive low-scoring creatives, maintain Top 5 per SKU, maintain Elite Top 20. Add route `POST /api/agent/library-cleanup` for on-demand runs.

**Deliverable:** Creative library stays clean and high-quality. Assembly workspace only shows the best options.

---

### 3.6 Analytics Dashboard Section

Build the Analytics section in `app.js`. Pull data from `evics_renders`, `creatives`, `evics_trends`, and `publishing_queue`. Display: revenue trend, top creatives, channel breakdown, hook category performance, render success rate.

**Deliverable:** Full performance visibility without leaving the dashboard.

---

### 3.7 "Connect Sources" Modal

Build the in-app credential entry modal triggered by the "Connect Sources" button. Save to `localStorage`. Trigger live data hydration on save.

**Deliverable:** New users can connect Supabase without touching code or config files.

- Video generation success rate: target > 95%
- Creative approval rate: target > 70%
